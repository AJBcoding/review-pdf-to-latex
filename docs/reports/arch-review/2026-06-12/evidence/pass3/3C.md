### Abandoned-round tombstone is re-emitted by the results watcher (contract bug)   [severity: high] [effort: S]
- FACT (cited): `abandonRound` renames a results file to the `.abandoned.json`

  file, cited only to complete the bug chain): `abandonInterruptedRound`

- WHY IT MATTERS: clicking Abandon clears the round banner, then the
  debounced (120 ms, results-watcher.ts:46) re-emit of the renamed file

### `gt mail` stdin EPIPE can crash the Electron main process   [severity: med] [effort: S]
- FACT (cited): submit.ts:253-262 wraps `child.stdin.write(payload)` /
  `child.stdin.end()` in a synchronous try/catch; no error listener is ever
  attached to the stdin stream (verbatim command:
  `grep -n 'stdin.on\|stdin\.' desktop/main/submit.ts` â†’ only `254:
  child.stdin.write(payload);` and `255: child.stdin.end();`). The only
  error handler is on the child itself (`child.on('error', â€¦)`,
  submit.ts:278), which covers spawn failure, not stream errors.
- WHY IT MATTERS: if `gt` exits before draining stdin (bad subcommand,
  version mismatch, immediate usage error), Node emits an asynchronous
  EPIPE on the stdin stream; with no `'error'` listener that becomes an
  uncaught exception in the main process â€” the synchronous try/catch does
  not catch it. JUDGEMENT: low-frequency but worst-case blast radius (whole
  app), in the one pipeline that shells out per user action.
- RECOMMENDATION: add `child.stdin.on('error', â€¦)` routing into the existing
  `settle({ ok:false, reason:'gt_failed', â€¦ })` path before writing. S.

### Bundle writer is anchor-kind-blind â€” assumes PDF coordinates on every comment   [severity: med] [effort: M]
- FACT (cited): `buildHighlight` reads `comment.anchor.region` with no
  discriminator check (bundle.ts:67-68 `const r = comment.anchor.region;`),
  and page grouping reads `c.anchor.page` (bundle.ts:203-204). The data
  model forces this: `CommentPayload.anchor: AnchorRegion` is a REQUIRED,
  PDF-shaped field (shared/types.ts:130, with AnchorRegion = page +
  x/y/w/h at types.ts:121-124) on every comment in every format, while md
  comments carry their real anchor in the optional `md_anchor` field
  (types.ts:157-165) keyed off a file-level `anchor_kind` discriminator
  (types.ts:168-170, 190).
- WHY IT MATTERS: the unified cross-format comment model is the charter's
  capability centerpiece. Today only renderer call-site discipline prevents
  an md/html/docx comment (whose `anchor` is a placeholder) from flowing
  into `writeBundle` and producing a syntactically valid PDF with nonsense
  /Rect + /QuadPoints. Any DOCX comments.xml writer added beside this will
  face the same untyped seam.
- RECOMMENDATION: make the anchor a discriminated union per comment (not a
  file-level flag plus parallel optional fields), and have `writeBundle`
  narrow to the pdf anchor kind, skipping/reporting others â€” this is the
  natural first brick of the unified model and the per-format adapter
  layer. M (type change ripples through renderer + sidecars; schema_version
  bump or tolerant reader required).
